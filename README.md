# 실시간성 게임 랭킹 시스템 개선 프로젝트

## 프로젝트 개요
- 랭킹 시스템을 구현합니다.
- RDBMS 로 구현된 랭킹 시스템이 부하를 얼마나 견디는지 확인해보고 개선하는 프로젝트입니다.
- 마지막엔 Redis sorted set 을 사용하여 개선합니다.

## 프로젝트 목표
- 실시간성이 중요한 시스템에서 RDBMS의 병목을 파악하고 Redis sorted set을 도입하여 DB 병목을 개선하는 것을 목표로합니다.

## 실험 환경
실험은 AWS EC2 를 통한 클라우드 환경에서 진행하였습니다.

인스턴스 
- t3.micro
- 메모리 1GB
- 스왑 메모리 1GB

Database
- MySQL

## 프로젝트를 통한 인사이트
- 순위 조회 과정에서 RDBMS 와 Redis 의 실제 성능 차이를 느낄 수 있었습니다.
  - 레디스가 빠르니까 쓰자는 말보다. 현재 트래픽에서 데이터가 5만 건이 넘어가면 응답 시간이 1초를 초과하므로 레디스를 씁시다. 라고 근거를 확실하게 말할 수 있게되었습니다.
- 단순 DB 쿼리를 던지는 것과 특정 목적에 최적화된 자료구조(sorted set)를 쓰는 것의 차이를 확인할 수 있었습니다.
